### Streaming Gemini Responses

By default `generate_content` waits for the full response. Use `stream=True` to receive tokens as they are generated — essential for a good user experience in chat interfaces.

```python
import google.generativeai as genai

genai.configure(api_key="AIzaSy...")
model = genai.GenerativeModel("gemini-1.5-flash")

# Streaming — tokens arrive incrementally
response = model.generate_content(
    "Write a detailed market summary for UAE equities in 2026.",
    stream=True,
)

for chunk in response:
    print(chunk.text, end="", flush=True)   # prints token by token

print()  # newline after stream ends

# resolve() consolidates streamed chunks — access full response afterwards
response.resolve()
print(response.usage_metadata)   # token counts available after resolve()
```

**FastAPI streaming endpoint with Gemini:**

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import google.generativeai as genai

app = FastAPI()
genai.configure(api_key="AIzaSy...")
model = genai.GenerativeModel("gemini-1.5-flash")

async def stream_gemini(prompt: str):
    response = model.generate_content(prompt, stream=True)
    for chunk in response:
        if chunk.text:
            yield chunk.text

@app.get("/stream")
async def stream_endpoint(prompt: str):
    return StreamingResponse(stream_gemini(prompt), media_type="text/plain")
```

### Generating Text from Images (Vision)

Gemini is natively multimodal — you can pass images alongside text in the same `generate_content` call. Gemini reads and reasons about images without any extra setup.

**Option 1 — Inline image from local file (using PIL):**

```python
import google.generativeai as genai
from PIL import Image

genai.configure(api_key="AIzaSy...")
model = genai.GenerativeModel("gemini-1.5-pro")

image = Image.open("fund_performance_chart.png")

response = model.generate_content([
    "Analyse this fund performance chart. What trend do you see, and are there any anomalies?",
    image,
])
print(response.text)
```

**Option 2 — Base64 encoded image (no PIL needed):**

```python
import google.generativeai as genai
import base64

genai.configure(api_key="AIzaSy...")
model = genai.GenerativeModel("gemini-1.5-pro")

with open("portfolio_screenshot.png", "rb") as f:
    image_data = base64.b64encode(f.read()).decode("utf-8")

response = model.generate_content([
    {
        "inline_data": {
            "mime_type": "image/png",
            "data": image_data,
        }
    },
    "Extract all fund names and their current NAV values from this screenshot.",
])
print(response.text)
```

**Option 3 — Image via URL (Gemini 1.5+ only):**

```python
response = model.generate_content([
    {
        "file_data": {
            "mime_type": "image/jpeg",
            "file_uri": "https://example.com/annual_report_chart.jpg",
        }
    },
    "Summarise the key financial figures shown in this chart.",
])
```

**Supported MIME types:** `image/png`, `image/jpeg`, `image/gif`, `image/webp`, `video/mp4`, `audio/mp3`, `application/pdf`

## 2. Gemini Generation Config

`GenerationConfig` groups all the sampling and output control parameters into a single object you pass to `GenerativeModel` or `generate_content`.

```python
import google.generativeai as genai
from google.generativeai.types import GenerationConfig

genai.configure(api_key="AIzaSy...")

generation_config = GenerationConfig(
    candidate_count=1,       # number of response candidates to generate
    stop_sequences=["\n\n", "END"],  # stop generating at any of these strings
    max_output_tokens=512,   # cap on output tokens
    temperature=0.4,         # randomness (0.0 = deterministic, 2.0 = very random)
    top_p=0.95,              # nucleus sampling — consider tokens summing to top 95% probability
    top_k=40,                # only sample from the top 40 most probable next tokens
)

model = genai.GenerativeModel(
    model_name="gemini-1.5-pro",
    generation_config=generation_config,
)

response = model.generate_content("Explain portfolio rebalancing in simple terms.")
print(response.text)
```

You can also override config per individual call:

```python
# Override just for this one call
response = model.generate_content(
    "Write a creative tagline for a robo-advisor app.",
    generation_config=GenerationConfig(temperature=1.2, max_output_tokens=50),
)
```

---

### Parameters Explained

#### `candidate_count`
Number of independent response candidates to generate. Unlike OpenAI's `n`, Gemini's `candidate_count` is currently limited to `1` for most models. Reserved for future use with multi-candidate selection.

```python
GenerationConfig(candidate_count=1)   # only supported value in production
```

---

#### `stop_sequences`
A list of strings that will halt generation as soon as any one of them appears in the output. The stop string itself is excluded from the response.

```python
GenerationConfig(stop_sequences=["\n\n", "STOP", "---"])
# Model stops at the first double newline, the word "STOP", or "---"
```

Useful for: structured generation where you want exactly one paragraph, SQL generation (stop at `;`), dialogue systems (stop at `"Human:"`).

---

#### `max_output_tokens`
Maximum tokens the model can generate. Does not include input tokens. If the model hits this limit mid-sentence, the response is cut off.

```python
GenerationConfig(max_output_tokens=256)   # ~200 words of output
```

---

#### `temperature`
Controls randomness in token selection. Range: `0.0` to `2.0`.

```python
GenerationConfig(temperature=0.0)    # fully deterministic — best for factual/structured output
GenerationConfig(temperature=0.7)    # balanced — general chat
GenerationConfig(temperature=1.4)    # creative — brainstorming, creative writing
```

---

#### `top_p` — Nucleus Sampling
Restricts the sampling pool to the smallest set of tokens whose cumulative probability is ≥ `top_p`. Controls diversity — lower = more focused.

```python
GenerationConfig(top_p=1.0)    # default — all tokens considered
GenerationConfig(top_p=0.85)   # only tokens in the top 85% probability mass
```

---

#### `top_k` — Top-K Sampling
At each step, only sample from the top `k` most probable next tokens. Specific to Gemini (OpenAI does not expose this parameter).

```python
GenerationConfig(top_k=1)     # always pick the single most likely token (greedy)
GenerationConfig(top_k=40)    # sample from the 40 most probable tokens (default-ish)
GenerationConfig(top_k=100)   # wider pool — more variety
```

**`top_k` vs `top_p`:**
| Parameter | How it filters | Advantage |
|-----------|---------------|-----------|
| `top_k` | Fixed number of candidates (e.g. always top 40 tokens) | Simple, predictable pool size |
| `top_p` | Variable — expands/shrinks pool based on probability distribution | Adapts to model confidence |

> Best practice: set `top_k` first to filter the long tail of unlikely tokens, then apply `top_p` to further refine within those candidates. Use both together for fine-grained control.

## 3. `generate_content` vs `start_chat`

These are the two interaction modes in the Gemini SDK. They serve different use cases.

### `generate_content` — Single-Turn / Stateless

Each call is **independent**. No conversation history is maintained between calls. You are responsible for managing any context you want to pass.

```python
import google.generativeai as genai

genai.configure(api_key="AIzaSy...")
model = genai.GenerativeModel("gemini-1.5-flash")

# Each call is a fresh, independent request
r1 = model.generate_content("What is a Sharpe ratio?")
print(r1.text)

r2 = model.generate_content("How do you calculate it?")
# ⚠️ Model has NO memory of r1 — "it" refers to nothing
print(r2.text)   # likely a generic or confused answer
```

Use `generate_content` for:
- One-off tasks (summarisation, extraction, classification)
- Batch processing (process 500 documents independently)
- Any task where each request is self-contained

---

### `start_chat` — Multi-Turn / Stateful

`start_chat` creates a **chat session** that automatically accumulates conversation history. Every subsequent `send_message` call includes all prior turns — the model has full context.

```python
import google.generativeai as genai

genai.configure(api_key="AIzaSy...")
model = genai.GenerativeModel(
    model_name="gemini-1.5-pro",
    system_instruction="You are a UAE wealth advisor. Be concise and cite sources.",
)

chat = model.start_chat(history=[])   # start fresh, or pass existing history

# Turn 1
r1 = chat.send_message("What is a Sharpe ratio?")
print(r1.text)

# Turn 2 — model remembers turn 1
r2 = chat.send_message("How do you calculate it?")
print(r2.text)   # ✅ "it" correctly refers to Sharpe ratio

# Turn 3
r3 = chat.send_message("What is a good Sharpe ratio for a UAE equity fund?")
print(r3.text)   # ✅ full context of prior turns available

# Inspect the full conversation history
for msg in chat.history:
    print(f"{msg.role}: {msg.parts[0].text[:80]}")
```

**Pre-loading history** (useful for resuming a saved conversation):

```python
history = [
    {"role": "user",  "parts": ["What funds do you recommend for a moderate risk profile?"]},
    {"role": "model", "parts": ["For a moderate risk profile, I'd suggest a blend of..."]},
]

chat = model.start_chat(history=history)
# Continue from where the previous session left off
r = chat.send_message("What about for a conservative profile?")
```

---

### `generate_content` vs `start_chat` — Side-by-Side

| Feature | `generate_content` | `start_chat` / `send_message` |
|---------|--------------------|-------------------------------|
| State | Stateless — each call independent | Stateful — history auto-managed |
| History | You pass it manually if needed | SDK appends each turn automatically |
| Use case | Batch, one-off, self-contained | Conversational chat, multi-turn Q&A |
| Streaming | `stream=True` | `send_message(..., stream=True)` |
| System instruction | Set on model | Set on model, applies to all turns |
| Cost | Pay per call | Pay per call (all history tokens re-sent each turn) |

> **Cost note:** In `start_chat`, the full conversation history is re-sent on every turn. A 20-turn conversation sends ~20x more tokens than turn 1. Use `generate_content` with manual context trimming for cost-sensitive long conversations.

## 4. Model Info with `get_model` — Token Limits, Temperature, top_p, top_k

Use `genai.get_model()` to introspect a model's capabilities and constraints before using it — especially useful when switching between models or building model-agnostic code.

```python
import google.generativeai as genai

genai.configure(api_key="AIzaSy...")

model_info = genai.get_model("models/gemini-1.5-pro")

print(model_info.name)                    # models/gemini-1.5-pro
print(model_info.display_name)            # Gemini 1.5 Pro
print(model_info.description)

# Token limits
print(model_info.input_token_limit)       # e.g. 1_048_576 (1M tokens)
print(model_info.output_token_limit)      # e.g. 8_192

# Default and supported parameter ranges
print(model_info.temperature)             # default temperature
print(model_info.top_p)                   # default top_p
print(model_info.top_k)                   # default top_k

# Supported generation methods
print(model_info.supported_generation_methods)   # ['generateContent', 'countTokens']
```

**List all available models:**

```python
for m in genai.list_models():
    if "generateContent" in m.supported_generation_methods:
        print(f"{m.name:40} input={m.input_token_limit:>8,}  output={m.output_token_limit:>6,}")
```

Example output:
```
models/gemini-1.5-pro-latest         input=1,048,576  output= 8,192
models/gemini-1.5-flash-latest       input=1,048,576  output= 8,192
models/gemini-2.5-pro-preview        input=1,048,576  output=65,536
```

**Practical use — guard against exceeding context:**

```python
model_info = genai.get_model("models/gemini-1.5-flash")
max_input = model_info.input_token_limit

# Before sending a large document, check it fits
token_count = model.count_tokens(large_document_text).total_tokens
if token_count > max_input * 0.9:   # leave 10% headroom
    raise ValueError(f"Document too large: {token_count} tokens (limit {max_input})")
```

## 5. `count_tokens`

Before sending a request, you can count exactly how many tokens your prompt will consume — without actually making a generation call. Essential for cost estimation, context window management, and avoiding `429 token limit exceeded` errors.

```python
import google.generativeai as genai

genai.configure(api_key="AIzaSy...")
model = genai.GenerativeModel("gemini-1.5-pro")

# Count tokens in a simple text prompt
result = model.count_tokens("What is the NAV of the ENBD Islamic Fund?")
print(result.total_tokens)   # e.g. 14

# Count tokens in a multi-part prompt (text + image)
from PIL import Image
image = Image.open("annual_report.png")

result = model.count_tokens([
    "Summarise the key financial figures from this annual report.",
    image,
])
print(result.total_tokens)   # text tokens + image tokens (images ≈ 258 tokens per 256x256 tile)

# Count tokens in a chat history before sending
chat = model.start_chat(history=[])
chat.send_message("What is a bond?")
chat.send_message("How do bond yields relate to prices?")

# Count the full accumulated history
result = model.count_tokens(chat.history)
print(result.total_tokens)   # total tokens across all turns so far
```

**Estimate cost before a large batch job:**

```python
import google.generativeai as genai

genai.configure(api_key="AIzaSy...")
model = genai.GenerativeModel("gemini-1.5-flash")

documents = [doc1_text, doc2_text, doc3_text, ...]   # list of strings to process

total_input_tokens = 0
for doc in documents:
    result = model.count_tokens(doc)
    total_input_tokens += result.total_tokens

# Gemini 1.5 Flash pricing (example): $0.075 per 1M input tokens
cost_estimate = (total_input_tokens / 1_000_000) * 0.075
print(f"Total input tokens: {total_input_tokens:,}")
print(f"Estimated input cost: ${cost_estimate:.4f}")
```

**After generation — actual token usage:**

```python
response = model.generate_content("Summarise UAE market conditions in 2026.")

# Actual usage (available after generation, not from count_tokens)
print(response.usage_metadata.prompt_token_count)       # input tokens used
print(response.usage_metadata.candidates_token_count)   # output tokens generated
print(response.usage_metadata.total_token_count)        # total = input + output
```

**`count_tokens` vs `usage_metadata` — when to use each:**
| Method | When to use |
|--------|------------|
| `model.count_tokens(prompt)` | **Before** sending — check if prompt fits, estimate cost |
| `response.usage_metadata` | **After** sending — get exact billed token counts |
