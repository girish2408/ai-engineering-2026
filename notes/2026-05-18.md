## 1. What is LangChain?

LangChain is an open-source Python (and JavaScript) framework for building applications powered by large language models. It provides composable building blocks — LLM wrappers, prompt templates, memory, tools, chains, and agents — so you don't wire everything from scratch for each project.

Think of it as the "Rails for LLM apps": opinionated, batteries-included, and built around the common patterns you'd otherwise rewrite every time (conversation memory, RAG pipelines, tool-calling agents).

```bash
pip install langchain langchain-openai langchain-community
```

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o", api_key="sk-...")
response = llm.invoke("What is a mutual fund?")
print(response.content)
```

## 2. How LangChain is Data-Aware and Agentic

### Data-Aware
A plain LLM only knows its training data. **Data-aware** means connecting the LLM to your own data — databases, PDFs, APIs, vector stores — so it can answer questions about *your* documents and information.

LangChain achieves this through:
- **Document loaders** — ingest PDFs, CSVs, websites, Notion pages
- **Text splitters** — chunk documents into pieces the LLM can process
- **Vector stores** — embed and store chunks for semantic retrieval (Qdrant, Chroma, FAISS)
- **Retrievers** — fetch the most relevant chunks at query time

```
User question → Retriever fetches relevant chunks → LLM reads chunks → Answers grounded in your data
```

### Agentic
**Agentic** means the LLM can take actions — it's not just answering questions but deciding *what to do next*. The LLM reasons about which tool to use, calls it, reads the result, and decides whether to call another tool or return a final answer.

LangChain achieves this through:
- **Tools** — functions the LLM can call (search, calculator, database query)
- **Agents** — LLM-driven loops that reason, act, observe, and repeat
- **Memory** — agents that remember past interactions across steps

```
Question → Agent reasons → calls Search tool → reads result → calls Calculator → returns answer
```

## 4. LangChain Main Concepts

LangChain is built around three layers:

```
┌──────────────────────────────────────────┐
│  AGENTS                                  │  ← LLM decides what to do next
│  (autonomous, tool-using loops)          │
├──────────────────────────────────────────┤
│  CHAINS                                  │  ← You decide the fixed sequence
│  (predefined sequences of steps)         │
├──────────────────────────────────────────┤
│  COMPONENTS                              │  ← The building blocks
│  (LLMs, prompts, memory, indexes, tools) │
└──────────────────────────────────────────┘
```

- **Components** are individual pieces — an LLM wrapper, a prompt template, a vector store retriever
- **Chains** combine components in a fixed sequence — always: retrieve → format → generate
- **Agents** use an LLM to dynamically decide the sequence — what to do and in what order is not hardcoded

### Indexes (Document Stores + Retrievers)
The machinery that makes LangChain data-aware. An index = document store + embedding model + vector DB.

```python
from langchain_community.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma

# 1. Load
loader = PyPDFLoader("fund_prospectus.pdf")
documents = loader.load()

# 2. Split
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(documents)

# 3. Embed + store
vectorstore = Chroma.from_documents(chunks, OpenAIEmbeddings())

# 4. Retrieve
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})
results = retriever.invoke("What is the fund's management fee?")
```

### Memory — Short-Term and Long-Term

**Short-term memory** keeps track of the current conversation — it's cleared when the session ends.

```python
from langchain.memory import ConversationBufferMemory

memory = ConversationBufferMemory(return_messages=True)
memory.save_context({"input": "My risk tolerance is moderate."}, {"output": "Noted."})
print(memory.load_memory_variables({}))
# {'history': [HumanMessage(content='My risk tolerance is moderate.'), AIMessage(content='Noted.')]}
```

**Long-term memory** persists across sessions — stored in a database or vector store.

```python
from langchain.memory import ConversationSummaryBufferMemory

# Summarises old messages to stay within context window — hybrid approach
memory = ConversationSummaryBufferMemory(
    llm=llm,
    max_token_limit=500,   # summarise once history exceeds 500 tokens
    return_messages=True,
)
```

| Memory Type | Storage | Persists? | Best for |
|-------------|---------|-----------|---------|
| `ConversationBufferMemory` | In-memory list | No | Short chats |
| `ConversationSummaryBufferMemory` | In-memory + LLM summary | No | Long chats, context window management |
| `ConversationTokenBufferMemory` | Last N tokens | No | Token-budget-aware chats |
| `VectorStoreRetrieverMemory` | Vector DB | Yes | Semantic long-term recall |

## 6. LangChain Chains

Chains combine components into a fixed, repeatable sequence. The classic example: **retrieve context → format prompt → call LLM → parse output**.

The modern way to build chains is **LCEL** (LangChain Expression Language) — see section 11. The older way used pre-built chain classes (e.g. `RetrievalQA`, `ConversationChain`) which are now largely superseded by LCEL.

**Why chains?**
- Reusable pipelines — define once, call many times
- Composable — chain outputs feed into chain inputs
- Traceable — LangSmith can trace every step

## 7. LangChain Agents

Agents give the LLM **autonomy** — instead of you defining the steps, the LLM decides which tools to call and in what order.

**Chain vs Agent:**
| | Chain | Agent |
|-|-------|-------|
| Sequence | Fixed — you define it | Dynamic — LLM decides |
| Tools | Not used | Core capability |
| Best for | Predictable pipelines | Open-ended tasks requiring reasoning |
| Reliability | High — deterministic | Lower — LLM can make wrong tool choices |

```python
from langchain.agents import create_react_agent, AgentExecutor
from langchain_community.tools import DuckDuckGoSearchRun, WikipediaQueryRun

tools = [DuckDuckGoSearchRun(), WikipediaQueryRun()]
# Agent decides: "I need to search for this", calls DuckDuckGo, reads result, decides next step
```

## 8. Caching in LangChain

LLM API calls are slow and expensive. Caching avoids calling the LLM again for identical inputs — the second time you ask the same question, the cached answer is returned instantly for free.

### In-Memory Cache
Fast, zero setup. Lost when the process restarts. Good for development.

```python
from langchain.globals import set_llm_cache
from langchain.cache import InMemoryCache
from langchain_openai import ChatOpenAI
import time

set_llm_cache(InMemoryCache())
llm = ChatOpenAI(model="gpt-4o")

# First call — hits the API
t0 = time.time()
r1 = llm.invoke("What is a mutual fund?")
print(f"First call: {time.time()-t0:.2f}s")   # e.g. 1.8s

# Second call — served from cache
t0 = time.time()
r2 = llm.invoke("What is a mutual fund?")
print(f"Second call: {time.time()-t0:.4f}s")  # e.g. 0.0002s — instant
```

### SQLite Cache
Persists to disk — cache survives restarts. Great for development and low-traffic production.

```python
from langchain.cache import SQLiteCache
from langchain.globals import set_llm_cache

set_llm_cache(SQLiteCache(database_path=".langchain_cache.db"))
llm = ChatOpenAI(model="gpt-4o")

# Cached responses are stored in .langchain_cache.db
# Restart the app — cache still works
r = llm.invoke("Explain portfolio diversification.")
```

**When caching helps most:**
- FAQ bots — many users ask the same questions
- Batch document processing with repeated prompts
- Development and testing — avoid paying for repeated test runs

**When NOT to cache:**
- Personalised responses (each user's context is different)
- Real-time data queries (answer will change)
- Creative tasks where variety is desired

## 9. LLM Streaming in LangChain

Streaming returns tokens as they are generated rather than waiting for the full response.

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o", streaming=True)

# Method 1 — stream() iterator
for chunk in llm.stream("Write a summary of UAE equity market trends in 2026."):
    print(chunk.content, end="", flush=True)

print()  # final newline
```

**With callbacks (useful for logging / UI updates):**

```python
from langchain.callbacks.streaming_stdout import StreamingStdOutCallbackHandler

llm = ChatOpenAI(
    model="gpt-4o",
    streaming=True,
    callbacks=[StreamingStdOutCallbackHandler()],   # auto-prints each token
)

llm.invoke("Explain compound interest.")   # tokens print as they arrive
```

**Streaming in a chain:**

```python
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template("Summarise: {text}")
chain = prompt | llm

for chunk in chain.stream({"text": "ENBD annual report excerpt..."}):
    print(chunk.content, end="", flush=True)
```

## 10. Prompt Templates — Dynamic Prompts

### Why Use Prompt Templates Instead of Plain f-strings?

Good question. At first glance a template seems unnecessary — why not just use an f-string?

```python
# Plain f-string — works, but...
prompt = f"Summarise this document in {num_words} words: {document}"
llm.invoke(prompt)
```

Prompt templates give you:
1. **Separation of concerns** — the prompt structure lives separately from the values
2. **Reusability** — define once, call with different inputs across your whole app
3. **Composability** — templates chain together in LCEL pipelines (`prompt | llm | parser`)
4. **Validation** — LangChain checks that all required variables are provided before sending
5. **Serialisation** — save/load prompt templates as JSON/YAML for version control
6. **LangSmith tracing** — templates are traced individually, so you can see exactly what was sent

```python
from langchain_core.prompts import PromptTemplate

# Define once — the structure
template = PromptTemplate(
    template="You are a {role}. {task} Use a {tone} tone and keep it under {max_words} words.",
    input_variables=["role", "task", "tone", "max_words"],
)

# Reuse with different values
p1 = template.format(role="financial advisor", task="Explain bond yields.", tone="formal", max_words="100")
p2 = template.format(role="teacher", task="Explain bond yields to a 12-year-old.", tone="simple", max_words="50")

# Or use in a chain — values injected at runtime
chain = template | llm
result = chain.invoke({"role": "analyst", "task": "Summarise Q1 earnings.", "tone": "concise", "max_words": "80"})
```

**Real-world use case:** You have 500 fund documents to process. You define one template and loop over documents — no copy-pasting, no f-string bugs, full tracing of what was sent for each.

```python
summarise_template = PromptTemplate(
    template="""Summarise the following fund document.
Focus on: fees, risk level, and historical performance.
Document: {document}
Output as JSON: {{"summary": "", "fees": "", "risk": "", "performance": ""}}""",
    input_variables=["document"],
)

chain = summarise_template | llm
for doc in fund_documents:
    result = chain.invoke({"document": doc})
```

## 11. ChatPromptTemplate — Why Different from PromptTemplate?

`PromptTemplate` produces a **single string** — the whole prompt is one piece of text. This is fine for older completion-style LLMs.

`ChatPromptTemplate` produces a **list of messages** with roles (`system`, `human`, `ai`). This matches how chat models (GPT-4o, Claude, Gemini) actually work — they expect structured role-separated messages, not a flat string.

```python
from langchain_core.prompts import PromptTemplate, ChatPromptTemplate

# PromptTemplate — produces a single string
pt = PromptTemplate.from_template("Summarise this document: {document}")
print(pt.format(document="..."))
# Output: "Summarise this document: ..."
# → sent as a single user message

# ChatPromptTemplate — produces structured messages
cpt = ChatPromptTemplate.from_messages([
    ("system", "You are a wealth advisor specialising in {domain}."),
    ("human",  "Explain {concept} to a {audience}."),
])

messages = cpt.format_messages(domain="UAE equities", concept="bond yield", audience="first-time investor")
# Output:
# [SystemMessage(content='You are a wealth advisor specialising in UAE equities.'),
#  HumanMessage(content='Explain bond yield to a first-time investor.')]
```

**Including conversation history in ChatPromptTemplate:**

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

cpt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful financial assistant."),
    MessagesPlaceholder(variable_name="history"),   # ← injects full chat history here
    ("human", "{input}"),
])

# At runtime, history is the list of prior messages from memory
chain = cpt | llm
result = chain.invoke({"input": "What's my risk profile?", "history": memory.chat_memory.messages})
```

**Use `PromptTemplate` when:** calling a completion-style endpoint or building a simple one-shot prompt.
**Use `ChatPromptTemplate` when:** working with chat models and needing system prompts, multi-turn history, or role-separated messages — which is almost always in practice.

## 12. LCEL — LangChain Expression Language

LCEL is the modern way to compose chains using the `|` (pipe) operator. It works like Unix pipes: the output of one component becomes the input of the next.

```
prompt | llm | output_parser
```

Each `|` step transforms the data. The whole chain becomes a single callable object you can `.invoke()`, `.stream()`, or `.batch()`.

### Basic Chain

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

llm = ChatOpenAI(model="gpt-4o")
parser = StrOutputParser()   # extracts .content string from AIMessage

chain = (
    ChatPromptTemplate.from_template("Translate to French: {text}")
    | llm
    | parser
)

result = chain.invoke({"text": "The market is bullish today."})
print(result)   # "Le marché est haussier aujourd'hui."
```

### RAG Chain

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_community.vectorstores import Chroma
from langchain_core.output_parsers import StrOutputParser

# Setup (done once)
vectorstore = Chroma.from_documents(chunks, OpenAIEmbeddings())
retriever   = vectorstore.as_retriever(search_kwargs={"k": 4})

prompt = ChatPromptTemplate.from_template("""
Answer the question using only the context below.
If the answer is not in the context, say "I don't know."

Context: {context}
Question: {question}
""")

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | ChatOpenAI(model="gpt-4o")
    | StrOutputParser()
)

answer = rag_chain.invoke("What is the management fee for the ENBD Islamic Fund?")
```

### Chaining Two LLM Calls

```python
# Chain 1: summarise
summarise_chain = (
    ChatPromptTemplate.from_template("Summarise this in 3 bullet points: {document}")
    | llm
    | StrOutputParser()
)

# Chain 2: translate the summary
translate_chain = (
    ChatPromptTemplate.from_template("Translate to Arabic: {summary}")
    | llm
    | StrOutputParser()
)

# Compose — output of summarise feeds into translate
full_chain = summarise_chain | (lambda summary: translate_chain.invoke({"summary": summary}))
result = full_chain.invoke({"document": large_document})
```

### Batch Processing

```python
# Process multiple inputs in parallel — LCEL handles concurrency
questions = [
    {"question": "What is NAV?"},
    {"question": "What is an ETF?"},
    {"question": "What is a bond?"},
]
results = rag_chain.batch(questions)   # runs all in parallel
```

**LCEL advantages:**
- Streaming works automatically — `chain.stream(input)` streams through every step
- Batching works automatically — `chain.batch([...])` runs in parallel
- Every step is traced in LangSmith without extra code
- Async works automatically — `await chain.ainvoke(input)` for FastAPI

## 13. LangChain Tools

Tools expand what an LLM can do beyond generating text. A tool is a function with a name, description, and typed inputs — the LLM reads the description to decide when to call it.

```python
from langchain_community.tools import DuckDuckGoSearchRun, WikipediaQueryRun
from langchain_community.utilities import WikipediaAPIWrapper

search = DuckDuckGoSearchRun()
wiki   = WikipediaQueryRun(api_wrapper=WikipediaAPIWrapper())

# Use a tool directly
result = search.invoke("ENBD stock price today")
print(result)

result = wiki.invoke("Sharpe ratio")
print(result)
```

### Custom Tool

```python
from langchain.tools import tool

@tool
def get_fund_nav(isin: str) -> str:
    """Get the current Net Asset Value for a fund by its ISIN code."""
    # In real life: call your internal API
    nav_data = {"IE00B4L5Y983": "AED 12.45", "LU0011850392": "AED 8.90"}
    return nav_data.get(isin, f"NAV not found for ISIN {isin}")

print(get_fund_nav.name)         # "get_fund_nav"
print(get_fund_nav.description)  # the docstring — what the LLM reads to decide when to call it
```

**The docstring is the most important part** — it tells the LLM *when* and *how* to use the tool. Write it clearly and specifically.

## 14. ReAct Agent — Reasoning and Acting

### What is ReAct?

**ReAct = Reasoning + Acting**. It is an agent pattern where the LLM alternates between:
1. **Reasoning** (Thought) — thinking about what to do next
2. **Acting** (Action) — calling a tool
3. **Observing** (Observation) — reading the tool's result
4. Repeating until it has a final answer

This is a form of **chain-of-thought prompting** — the model's reasoning is made explicit in the output before each action, which improves accuracy and makes the agent's behaviour auditable.

```
Thought: I need to find the current price of ENBD shares.
Action: duckduckgo_search
Action Input: "ENBD stock price today"
Observation: Emirates NBD shares closed at AED 17.40 on May 18 2026...

Thought: I have the price. Now I need to calculate the return.
Action: python_repl
Action Input: (17.40 - 15.00) / 15.00 * 100
Observation: 16.0

Thought: I now have all the information needed.
Final Answer: ENBD shares are at AED 17.40, representing a 16% return from AED 15.00.
```

---

### LangChain Hub — Pulling Prompts

The **LangChain Hub** is a registry of community-contributed prompts. Instead of writing the ReAct system prompt from scratch, you pull a battle-tested one.

```python
from langchain import hub

# Pull the standard ReAct prompt — written by Harrison Chase (hwchase17)
prompt = hub.pull("hwchase17/react")
print(prompt.template)
```

**Why pull from the hub instead of writing your own?**
- The `hwchase17/react` prompt has been iterated on extensively — it reliably elicits the `Thought / Action / Action Input / Observation` format
- Writing your own ReAct prompt from scratch and getting it right takes many iterations
- You can always customise it after pulling it: `prompt.partial(instructions="Always respond in English")`

### Full ReAct Agent Example

```python
from langchain import hub
from langchain_openai import ChatOpenAI
from langchain.agents import create_react_agent, AgentExecutor
from langchain_community.tools import DuckDuckGoSearchRun, WikipediaQueryRun
from langchain_community.utilities import WikipediaAPIWrapper
from langchain_experimental.tools import PythonREPLTool
from langchain.tools import tool

# 1. Define tools
@tool
def get_fund_nav(isin: str) -> str:
    """Get the current Net Asset Value for a UAE fund given its ISIN code.
    Use this when the user asks about fund prices or NAVs."""
    nav_data = {"IE00B4L5Y983": "AED 12.45", "LU0011850392": "AED 8.90"}
    return nav_data.get(isin, f"NAV not found for {isin}")

tools = [
    DuckDuckGoSearchRun(),                                      # web search
    WikipediaQueryRun(api_wrapper=WikipediaAPIWrapper()),        # Wikipedia
    PythonREPLTool(),                                           # run Python code
    get_fund_nav,                                               # custom tool
]

# 2. LLM
llm = ChatOpenAI(model="gpt-4o", temperature=0)

# 3. Pull the ReAct prompt from LangChain Hub
prompt = hub.pull("hwchase17/react")

# 4. Create the agent
agent = create_react_agent(
    llm=llm,
    tools=tools,
    prompt=prompt,
)

# 5. Wrap in AgentExecutor
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,                    # prints Thought/Action/Observation at each step
    handle_parsing_errors=True,      # recovers if LLM output doesn't match expected format
    max_iterations=8,                # safety limit — stop after 8 reasoning steps
    max_execution_time=60,           # safety limit — stop after 60 seconds
    return_intermediate_steps=True,  # include all steps in the response
)

# 6. Run
result = executor.invoke({
    "input": "Search for the current price of ENBD shares, then calculate what a AED 10,000 investment made at AED 15.00 per share would be worth today."
})

print(result["output"])

# Access intermediate steps (Thought/Action/Observation tuples)
for step in result["intermediate_steps"]:
    action, observation = step
    print(f"Tool called: {action.tool}")
    print(f"Tool input:  {action.tool_input}")
    print(f"Observation: {observation[:200]}")
    print()
```

---

### How the Agent Chooses Which Tool to Use

The agent reads the **name and description of each tool** and reasons about which one matches the task. This is why tool descriptions are critical.

```python
# The agent sees this "menu" of tools:
# - duckduckgo_search: "A wrapper around DuckDuckGo Search. Useful for when you need to answer
#                       questions about current events or search the web."
# - wikipedia: "A wrapper around Wikipedia. Useful for when you need general background
#               knowledge about a topic."
# - python_repl: "A Python shell. Use this to execute python commands. Input should be a valid
#                 python command."
# - get_fund_nav: "Get the current Net Asset Value for a UAE fund given its ISIN code.
#                  Use this when the user asks about fund prices or NAVs."
```

For the query *"Search for the current price of ENBD shares"*:
- Agent reasons: "I need current market data" → picks `duckduckgo_search` (not Wikipedia — that's for background knowledge, not live prices)

For *"Calculate the return"*:
- Agent reasons: "I need to run a calculation" → picks `python_repl`

---

### `AgentExecutor` — Key Parameters

```python
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,
    # Error handling
    handle_parsing_errors=True,
    # "If the LLM output doesn't match the Thought/Action/Observation format,
    #  send the malformed output back to the LLM as an error and ask it to fix it."
    # Without this, a single malformed output raises an exception and crashes the agent.

    # Safety limits — critical for production
    max_iterations=10,
    # Maximum number of Thought→Action→Observation loops.
    # Without this, a confused agent can loop forever (e.g. calling search repeatedly).

    max_execution_time=120,
    # Wall-clock timeout in seconds.
    # Stops the agent even if max_iterations isn't reached — protects against slow tools.

    return_intermediate_steps=True,
    # Include every Thought/Action/Observation in the response.
    # Essential for debugging and for LangSmith tracing.

    early_stopping_method="generate",
    # When max_iterations is hit: "generate" = ask LLM to produce a final answer from
    # whatever it has; "force" = return a fixed "Agent stopped" message.
)
```

**`handle_parsing_errors` in practice:**

```python
# Without handle_parsing_errors=True:
# If LLM outputs: "I'll search for that now." (missing the Action: format)
# → raises OutputParserException → agent crashes

# With handle_parsing_errors=True:
# LangChain sends back: "Your output was malformed. Please follow the format:
#  Thought: ...\nAction: ...\nAction Input: ..."
# → LLM corrects itself and continues
```

### Using Different Tools

```python
# Wikipedia — good for concepts, history, definitions
wiki = WikipediaQueryRun(api_wrapper=WikipediaAPIWrapper(top_k_results=2))

# DuckDuckGo — good for current events, prices, news
search = DuckDuckGoSearchRun()

# Tavily — better quality web search, structured results, built for AI agents
from langchain_community.tools.tavily_search import TavilySearchResults
tavily = TavilySearchResults(max_results=3, api_key="tvly-...")

# Python REPL — run arbitrary Python for calculations, data manipulation
```
