## 1. Jupyter Notebooks

A Jupyter Notebook is an interactive coding environment that runs in the browser. It lets you mix code, output, and markdown text in a single document called a "notebook" (`.ipynb` file). Each block of code is called a **cell** — you run cells individually, so you can experiment step by step without re-running your entire script.

```python
# Run a cell like this — output appears directly below it
print("Hello from a notebook cell")
2 + 2  # last expression auto-displays: 4
```

**Why it matters for AI engineering:** Notebooks are the standard for data exploration, model experimentation, and sharing reproducible results.

## 2. Virtual Environments (venv)

A virtual environment is an isolated Python installation scoped to a specific project. It keeps your project's dependencies separate from other projects and your system Python — so installing `langchain==0.1` for one project doesn't break another that needs `langchain==0.3`.

```bash
# Create a venv
python3 -m venv .venv

# Activate it (macOS/Linux)
source .venv/bin/activate

# Activate it (Windows)
.venv\Scripts\activate

# Install packages into it
pip install requests pandas

# Freeze dependencies for sharing
pip freeze > requirements.txt

# Deactivate when done
deactivate
```

**Rule of thumb:** one venv per project, always activate before coding.

## 3. Python Libraries

### 3.1 `requests` and `httpx`

Both libraries are used to make HTTP requests from Python. `requests` is synchronous (blocking); `httpx` supports async, making it better for AI apps that call multiple APIs concurrently.

```python
import requests

# Simple GET
response = requests.get("https://api.example.com/data")
print(response.status_code)   # 200
print(response.json())        # parsed JSON body

# POST with JSON payload
response = requests.post(
    "https://api.example.com/chat",
    json={"message": "Hello"},
)
```

```python
import httpx
import asyncio

# Async GET — non-blocking
async def fetch():
    async with httpx.AsyncClient() as client:
        r = await client.get("https://api.example.com/data")
        return r.json()

asyncio.run(fetch())
```

### 3.2 Handling HTTP Errors

Never assume a request succeeded. Always check the status code or raise on failure.

```python
import requests

response = requests.get("https://api.example.com/resource/999")

# Manual check
if response.status_code == 404:
    print("Not found")

# Or raise an exception for any 4xx/5xx automatically
response.raise_for_status()  # raises requests.HTTPError if status >= 400

# Full error handling pattern
try:
    r = requests.get("https://api.example.com/data", timeout=10)
    r.raise_for_status()
    data = r.json()
except requests.HTTPError as e:
    print(f"HTTP error: {e.response.status_code}")
except requests.ConnectionError:
    print("Network unreachable")
except requests.Timeout:
    print("Request timed out")
```

### 3.3 HTTP Authentication and Headers

APIs commonly require an API key or Bearer token sent in the request headers.

```python
import requests

API_KEY = "sk-abc123"

# Bearer token in Authorization header
headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json",
}

response = requests.get("https://api.openai.com/v1/models", headers=headers)

# Basic auth (username + password)
response = requests.get(
    "https://api.example.com/protected",
    auth=("username", "password"),
)
```

### 3.4 Pandas — Series, DataFrames, Import/Export

**Series** = a single column of data (1D). **DataFrame** = a table with rows and columns (2D).

```python
import pandas as pd

# Series
s = pd.Series([10, 20, 30], index=["a", "b", "c"])
print(s["b"])  # 20

# DataFrame
df = pd.DataFrame({
    "name":   ["Alice", "Bob", "Carol"],
    "salary": [90000, 75000, 110000],
})
```

**Importing and exporting:**

```python
# Import
df = pd.read_csv("data.csv")
df = pd.read_excel("data.xlsx")
df = pd.read_json("data.json")
df = pd.read_sql("SELECT * FROM users", con=engine)

# Export
df.to_csv("output.csv", index=False)
df.to_excel("output.xlsx", index=False)
df.to_json("output.json", orient="records")
df.to_sql("users", con=engine, if_exists="replace", index=False)

# YAML (pandas doesn't have native YAML — use PyYAML)
import yaml
with open("output.yaml", "w") as f:
    yaml.dump(df.to_dict(orient="records"), f)
```

**Adding and modifying columns/rows:**

```python
df["bonus"] = df["salary"] * 0.1           # new column
df["salary"] = df["salary"] + 5000         # modify column
df.loc[3] = ["Dave", 95000, 9500]           # add row by label
df = df.drop(index=3)                       # remove row
df = df.rename(columns={"salary": "base_pay"})
```

**`iloc` vs `loc`:**

```python
# iloc — integer-based position (0-indexed, like Python lists)
df.iloc[0]          # first row
df.iloc[0, 1]       # row 0, column 1
df.iloc[0:3]        # rows 0, 1, 2

# loc — label-based (uses index labels and column names)
df.loc[0]           # row with index label 0
df.loc[0, "name"]   # row 0, column "name"
df.loc[0:2]         # rows with labels 0 through 2 (inclusive!)
```

**Sampling and previewing:**

```python
df.head(5)     # first 5 rows
df.tail(5)     # last 5 rows
df.sample(3)   # 3 random rows (great for sanity checks)
df.shape       # (rows, columns)
df.dtypes      # data types of each column
```

### 3.5 Filtering Data

# Boolean mask
mask = df["salary"] > 80000
filtered = df[mask]

### 3.6 Handling Missing Data

```python
df.info()               # shows non-null counts per column — quick overview of missing data
df.isna()               # True/False mask of missing values
df.isna().sum()         # count of nulls per column

df.dropna()             # drop any row that has at least one NaN
df.dropna(axis=1)       # drop any column that has at least one NaN
df.dropna(subset=["salary"])  # drop rows where 'salary' is NaN only

df.fillna(0)            # replace all NaN with 0
df["salary"].fillna(df["salary"].mean())  # fill with column mean
df.fillna(method="ffill")  # forward fill — carry last valid value forward
```

### 3.7 Sorting Data

### 3.8 Aggregations and Groupby

```python
# Basic aggregations
df["salary"].sum()
df["salary"].mean()
df["salary"].max()

# groupby — split → apply → combine
df.groupby("dept")["salary"].sum()      # total salary per department
df.groupby("dept")["salary"].mean()     # average salary per department

# .agg() — multiple aggregations at once
df.groupby("dept")["salary"].agg(["sum", "mean", "count"])

# Multiple columns
df.groupby("dept").agg(
    total_salary=("salary", "sum"),
    headcount=("name", "count"),
)
```

### 3.9 Time Series Data Manipulation

```python
import pandas as pd

# to_datetime — convert a string column to datetime
df["date"] = pd.to_datetime(df["date"])          # auto-parse
df["date"] = pd.to_datetime(df["date"], format="%Y-%m-%d")  # explicit format

# Set date as index (needed for resample)
df = df.set_index("date")

# resample — bucket time series into periods
df["price"].resample("D").mean()   # daily average
df["price"].resample("W").sum()    # weekly total
df["price"].resample("M").last()   # monthly last value
df["price"].resample("Q").mean()   # quarterly average

# Example: daily stock prices → monthly average
prices = pd.Series(
    [100, 102, 98, 105, 110],
    index=pd.date_range("2026-01-01", periods=5, freq="B"),  # business days
)
monthly = prices.resample("M").mean()
print(monthly)
```

**Key frequency aliases:** `D` = day, `W` = week, `M` = month end, `MS` = month start, `Q` = quarter, `Y` = year.
